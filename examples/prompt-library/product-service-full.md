# Product or Service Presentation Prompt

Use this prompt to create a deck introducing a product, service, SaaS platform, tool, or solution.

```text
@presentation-pro-designer

Create a professional product/service PowerPoint presentation.

Presentation type:
Product or service presentation.

Product / service name:
[Insert name]

Purpose:
Create a clear presentation that explains what the product or service is, who it is for, how it works, and why it is valuable.

Audience:
[Clients / partners / internal stakeholders / users / prospects / investors]

Slide count:
[Insert number, or write: choose the best number based on content]

Source priority:
1. Follow this prompt.
2. Use attached product documentation, service description, screenshots, website copy, technical notes, or brochure as the main source.
3. Use attached design template or brand guide as the highest design priority.
4. Do not use external sources unless I approve.
5. Do not invent features, performance claims, pricing, guarantees, or integrations.

Required structure:
- Title slide
- Product/service promise
- Target audience or user problem
- Solution overview
- How it works
- Core features or service components
- Use cases
- Benefits and outcomes
- Implementation or onboarding process
- Proof points, screenshots, or examples if provided
- Pricing or packages, only if provided
- Next steps / CTA

Content requirements:
- Explain the product/service in plain English.
- Focus on user value and practical outcomes.
- Use examples and scenarios where useful.
- Use screenshots or diagrams only when they improve understanding.
- Avoid unsupported claims.

Design requirements:
- Modern, clean, product-focused, and brand-consistent.
- Use diagrams, feature cards, comparison tables, process flows, and screenshots where useful.
- Keep each slide focused on one idea.
- Use high-resolution visuals.
- Avoid clutter and tiny text.

Speaker notes:
Include presenter notes with talking points, suggested explanations, and optional demo cues.

Clarification rule:
If the audience, product maturity, features, pricing, or proof points are unclear, ask up to 5 specific questions before creating the deck.

Output:
- Create a PPTX file.
- Include a short summary of sources used, assumptions made, design decisions, and QA checks completed.
```
