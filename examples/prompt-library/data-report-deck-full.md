# Data Report Presentation Prompt

Use this prompt for data reports, performance reviews, dashboards, analytics summaries, operational updates, and results presentations.

```text
@presentation-pro-designer

Create a professional data report PowerPoint presentation.

Presentation type:
Data report / analytics presentation.

Report topic:
[Insert topic]

Audience:
[Leadership / department team / client / board / school leaders / operations team]

Purpose:
Create a clear presentation that turns the provided data into insights, charts, findings, risks, and recommended actions.

Slide count:
[Insert number, or write: choose the best number based on the data]

Source priority:
1. Follow this prompt.
2. Use attached spreadsheet, CSV, report, dashboard export, or data notes as the main source.
3. Use attached brand guide or PPTX template as the highest design priority.
4. Do not use external sources unless I approve.
5. Do not invent missing data.

Required structure:
- Title slide
- Executive summary
- Data source and scope
- Key metrics overview
- Trend analysis
- Comparison or segmentation analysis
- Main findings
- Risks, anomalies, or concerns
- Recommendations
- Next steps
- Appendix with tables or methodology

Chart requirements:
- Choose the simplest chart that fits the data.
- Use line charts for trends over time.
- Use bar or column charts for category comparison.
- Use scatter plots for relationships.
- Use tables when exact values matter.
- Avoid pie charts unless there are very few categories and part-to-whole comparison is essential.
- Use readable labels and direct annotations.
- Do not overload slides with multiple charts unless necessary.

Content requirements:
- Explain the insight, not just the data.
- Use action titles that state the conclusion.
- Highlight what changed, why it matters, and what should happen next.
- Separate facts from interpretation.
- Mark missing or unreliable data clearly.

Design requirements:
- Clean, professional, readable, and chart-focused.
- Follow any attached template.
- Use high contrast and large labels.
- Keep charts readable when projected.
- Avoid clutter and unnecessary decoration.

Speaker notes:
Include presenter notes explaining the interpretation of each chart and the key talking points.

Clarification rule:
If the data source, audience, metric definitions, reporting period, or decision context is unclear, ask up to 5 specific questions before creating the deck.

Output:
- Create a PPTX file.
- Include a short summary of sources used, assumptions made, chart choices, limitations, and QA checks completed.
```
