# Business Proposal Deck Prompt

Use this prompt for client proposals, service proposals, partnership proposals, or project proposals.

```text
@presentation-pro-designer

Create a professional business proposal PowerPoint presentation.

Presentation type:
Business proposal.

Proposal title:
[Insert proposal title]

Client / audience:
[Insert client name or audience type]

Purpose:
Create a persuasive but factual proposal deck that explains the client need, proposed solution, scope, deliverables, implementation approach, and next steps.

Slide count:
[Insert number, or write: choose the best number based on proposal complexity]

Source priority:
1. Follow this prompt.
2. Use attached client brief, RFP, notes, scope document, pricing file, or proposal draft as the main source.
3. Use attached brand guide or PPTX template as the highest design priority.
4. Do not use external sources unless I approve.
5. Do not invent pricing, dates, guarantees, legal terms, or client facts.

Required structure:
- Title slide
- Executive summary
- Client context or problem
- Goals and success criteria
- Proposed solution
- Scope of work
- Deliverables
- Timeline or implementation phases
- Team / capability / why us
- Pricing or options, only if provided
- Risks, dependencies, or assumptions
- Next steps / decision slide

Content requirements:
- Make the proposal clear, structured, and decision-ready.
- Link each solution element to the client need.
- Use benefits, not just features.
- Include tables for scope, deliverables, and timeline where useful.
- Clearly separate confirmed details from assumptions.
- Avoid vague claims like "world-class" or "cutting-edge" unless supported.

Design requirements:
- Professional, business-ready, clean, and brand-consistent.
- Use visual hierarchy, section dividers, process diagrams, and cards.
- Keep slides concise.
- Use high-quality visuals only.
- Ensure charts and tables are readable.

Speaker notes:
Include speaker notes that help the presenter explain each slide confidently.

Clarification rule:
If client needs, scope, timeline, budget, or audience are unclear, ask up to 5 specific questions before creating the deck.

Output:
- Create a PPTX file.
- Include a short summary of sources used, assumptions made, proposal risks, and QA checks completed.
```
