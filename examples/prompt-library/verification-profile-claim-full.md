# Verification / Profile Claim Deck Prompt

Use this prompt for company profile verification, domain ownership proof, profile claim review, platform verification, or data correction presentations.

```text
@presentation-pro-designer

Create a professional verification/profile claim PowerPoint presentation.

Presentation type:
Verification / profile claim presentation.

Entity / company name:
[Insert company name]

Purpose:
Create a factual presentation that supports profile ownership, domain recognition, official communication verification, and profile data correction.

Audience:
[Verification review team / platform profile team / business registry reviewer / partner compliance team]

Slide count:
[Insert number, or write: choose the best number based on source material]

Source priority:
1. Follow this prompt.
2. Use the attached verification pack, domain proof document, company document, emails list, website proof, or ownership statement as the main source.
3. Use the attached brand guide, logo, or PPTX template as the highest design priority.
4. Do not use external sources unless I approve.
5. Do not make legal, registration, ownership, or certification claims beyond what the attached source explicitly supports.

Required structure:
- Title slide
- Purpose of the verification pack
- Entity overview
- Company identity details
- Founder / owner / authorised representative, if provided
- Digital estate
- Official domains and subdomains
- Official email channels
- Public channels or social proof, if provided
- Verification methods available
- Limitations or clarification statement
- Closing verification statement

Content requirements:
- Use factual, careful, and precise wording.
- Clearly distinguish confirmed facts from in-progress or proposed details.
- If the source says registration is in progress, do not imply registration is complete.
- If a document is not a certificate, state that clearly.
- Avoid promotional language.
- Do not add unsupported claims.
- Use tables for identity, domains, emails, and verification methods.

Design requirements:
- Professional, clean, credible, and business-ready.
- Use a restrained corporate style.
- Follow any attached design template or brand identity.
- Use clear section headers, tables, cards, and verification icons.
- Avoid decorative visuals that weaken credibility.

Speaker notes:
Include speaker notes explaining how each slide supports verification.

Clarification rule:
If the platform, claim type, source authority, or verification goal is unclear, ask up to 5 specific questions before creating the deck.

Output:
- Create a PPTX file.
- Include a short summary of sources used, assumptions made, limitations, and QA checks completed.
```
