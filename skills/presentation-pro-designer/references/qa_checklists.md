# QA Checklists

## Source QA

- User files and templates were considered.
- External sources were not used unless approved.
- Missing data is marked as `DATA NEEDED`.
- Unsupported claims were removed or flagged.

## Content QA

- Deck type matches the request.
- Structure fits audience and purpose.
- Each slide has one clear job.
- Tone fits the audience.
- Speaker notes are included if requested.

## Design QA

- Template/brand guidance is followed.
- Typography is readable.
- Layouts are aligned and balanced.
- Colours meet practical contrast needs.
- Visual density is appropriate.

## Chart QA

- Chart type matches the data.
- Labels are readable.
- One key insight is clear.
- Data source is identified.
- No invented numbers are used.

## PPTX QA

- No text overflow.
- No overlapping objects.
- No stretched images.
- No broken placeholders.
- File opens and is ready to use.

## Delivery Summary

Include deck type, sources used, assumptions, design decisions, QA checks, and limitations.
