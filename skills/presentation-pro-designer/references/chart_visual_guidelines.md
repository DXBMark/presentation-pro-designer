# Chart and Data Visualization Guidelines

## Chart Selection

| Data Need | Preferred Visual |
|---|---|
| trend over time | line or area chart |
| category comparison | bar or column chart |
| rank comparison | sorted bar chart |
| part-to-whole | stacked bar, treemap, or simple pie/donut with few categories |
| distribution | histogram, boxplot, violin |
| correlation | scatter or bubble |
| process conversion | funnel |
| hierarchy | tree, org chart, treemap, mind map |
| relationship/flow | sankey, network, flow diagram |
| multidimensional comparison | radar |
| structured exact values | table or spreadsheet view |

## Slide Rules

- One chart should support one insight.
- Use direct labels where possible.
- Keep axis labels large enough for projection.
- Avoid unnecessary gridlines and legends.
- Do not use colour alone to encode meaning.
- Cite data sources in the deck summary or slide footer when required.
- Do not create charts from invented data.
