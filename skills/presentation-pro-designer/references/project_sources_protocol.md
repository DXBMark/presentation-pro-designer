# Project Sources Protocol

Project sources are files stored in a host workspace, project, repository, or connected app. They can include templates, brand guides, syllabi, standards, company profiles, financials, and sample decks.

## Rules

- Use project sources only when visible to the agent or explicitly provided.
- Do not claim to have used a project file unless it was accessible.
- If the user references a missing file, ask them to attach or identify it.
- Treat project templates and standards as primary context once visible.
- Summarize which project sources influenced the final plan or deck.

## Typical Project Sources

- `brand-guidelines.pdf`
- `company-profile.docx`
- `investor-template.pptx`
- `school-syllabus.pdf`
- `sample-deck.pptx`
- `financial-model.xlsx`
- `verification-pack.pdf`
