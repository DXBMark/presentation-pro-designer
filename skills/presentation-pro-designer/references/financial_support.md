# Financial Support for Investor Presentations

This file adapts financial-analysis skill concepts for presentation workflows. It does not provide live data by itself.

## Optional Data Sources

Use only when available and approved:
- uploaded financial statements, models, filings, investor decks, or spreadsheets
- host-provided finance tools
- Octagon MCP or equivalent financial data agent
- user-provided financial metrics

## Financial Modules

| Module | Use |
|---|---|
| income statement | revenue, gross profit, operating income, net income, EPS |
| balance sheet | assets, liabilities, equity, cash, net debt, liquidity |
| cash flow | operating cash flow, free cash flow, capex, cash conversion |
| financial growth | YoY growth across revenue, profit, EPS, and FCF |
| analyst estimates | consensus revenue/EPS, high-low range, revision trend |
| financial health scores | Altman Z-Score, Piotroski F-Score, distress/quality framing |
| peer comparison | growth, margins, multiples, returns, financial health |
| valuation | P/E, EV/EBITDA, EV/Revenue, scenario cases |

## Slide Mapping

| Slide Section | Financial Inputs |
|---|---|
| Executive snapshot | revenue, EPS, growth, estimates, health scores |
| Investment thesis | growth metrics, segment mix, margin trends |
| Financial performance | income statement + cash flow trends |
| Balance sheet strength | cash, debt, equity, liquidity, health scores |
| Peer comparison | growth, margins, multiples |
| Estimates | analyst estimates and operating assumptions |
| Valuation | forward metrics, multiples, scenario cases |
| Risks | leverage, FCF divergence, estimate uncertainty, ESG if supplied |

## Analysis Patterns

- Operating leverage: operating income growth exceeds revenue growth.
- Margin expansion: gross profit or operating income growth exceeds revenue growth.
- Margin compression: costs grow faster than revenue.
- EPS outperformance: EPS growth exceeds net income growth, possibly from buybacks.
- Earnings quality issue: net income growth exceeds FCF growth for multiple periods.

## Safety

Never fabricate financial data. If data is missing, write `DATA NEEDED` and suggest the required input, such as filings, financial model, analyst estimates, or tool access.
