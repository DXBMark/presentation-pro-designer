# Source Governance

## Source Priority

1. Current user prompt.
2. Uploaded files in the current task.
3. Visible project sources explicitly referenced by the user.
4. User-approved external sources/tools.
5. Skill reference guidance.

## External Sources

Do not use external websites, APIs, stock images, financial databases, or online references unless the user explicitly approves or requests that source. When external sources are unavailable, ask for files or mark gaps as `DATA NEEDED`.

## Attribution

For public repo usage and generated decks, keep source summaries clear:
- list user files used
- list external sources used if approved
- list assumptions
- avoid unsupported claims

## Financial Data

Financial and investor decks require verifiable data. Do not invent revenue, margins, estimates, valuation multiples, target prices, price charts, analyst ratings, or health scores.
